ML model & data pipeline

Predicting extreme
atmospheric events.

Two complementary models trained on 15 years of ERA5 reanalysis data (2010–2024) from a 1.5° × 1.5° grid centred on Seville. Binary target: will there be an extreme weather event tomorrow?

Model comparison

Both architectures were evaluated on the same temporal split. The Random Forest serves as the production predictor due to its superior AUC and interpretability. The LSTM was trained as a sequence-aware alternative using 21-day context windows.

Design decision

The LSTM was trained on only 9 raw ERA5 physical variables (no hand-crafted rolling features), making direct metric comparison unfair. Its lower F1 reflects both this constraint and the difficulty of predicting rare events with sequence models on ~15 years of daily data.

Random Forest — cross-validation

Time-series aware 5-fold cross-validation. Folds are strictly sequential — no data from the future leaks into any training window. The event rate is 12.2%, creating inherent class imbalance addressed with class_weight='balanced'.

Feature importance — top 10

Mean decrease in impurity across all trees. Precipitation total and cloud cover dominate — unsurprising for a binary target that includes rain events. Pressure gradient features rank consistently, confirming that synoptic-scale dynamics are the primary signal.

LSTM — training curve

Early stopping triggered at epoch 31 (patience 15 on AUC-PR). The model plateaued around epoch 16, suggesting the 9-feature set saturates relatively quickly. Loss continues decreasing while AUC-PR stagnates — a known signature of Focal Loss on imbalanced data.

LSTM architecture

A two-layer LSTM with LayerNorm and a GELU MLP head. LayerNorm was chosen over BatchNorm for stability with short sequences and small batch sizes. Focal Loss (α=0.75, γ=2) shifts gradient weight toward the rare positive class. 

# AtmosphericLSTM — forward pass
Input  →  (batch, 21, 9)          # 21-day window × 9 ERA5 features
LSTM₁  →  hidden=128               # short-range synoptic patterns
LSTM₂  →  hidden=64                # regime-level compression
LayerNorm  →  (batch, 64)          # last timestep only
Linear(64→32) + GELU + Dropout(0.25)
Linear(32→1)  + Sigmoid         # P(extreme event tomorrow)

Optimizer: AdamW (lr=3×10⁻⁴, weight decay=10⁻⁴) with cosine annealing. Gradient clipping at norm 1.0. Decision threshold optimised on validation F1: 0.43 (default 0.5 is sub-optimal under imbalance).

ERA5 data pipeline

01

Reanalysis download

CDS API (Copernicus). 7 variables at 06:00 / 12:00 / 18:00 UTC, 2010–2024. Area: 38°N–36.5°N, 6.5°W–5°W (bounding box around Seville). Format: NetCDF4, two file series — instantaneous and accumulated.

02

Physical conversion & daily aggregation

K → °C, Pa → hPa, precipitation × 1000 (m → mm), wind components → speed + direction. Humidity derived from dewpoint via Magnus formula. Resampled to daily: Tmax, Pmean, windmax, precipsum.

03

Bias correction

ERA5 systematically underestimates daily Tmax in Seville by ~2.5°C (confirmed against AEMET station data). A constant additive correction is applied before feature engineering and label thresholding.

04

Feature engineering (Random Forest)

32 features: raw daily aggregates, 3/7-day rolling means, first and third-order gradients, 7-day linear trend (polyfit), daily thermal range, pressure deficit vs. 7-day mean, sine/cosine seasonality encoding.

05

Label definition

Binary target: extreme event tomorrow (shift −1). Four triggers — heat (Tmax ≥ 38°C, AEMET ola de calor), cold (Tmax ≤ 10°C), wind (speed ≥ 8 m/s), rain (precip ≥ 0.5 mm). Event rate: 12.2%.
